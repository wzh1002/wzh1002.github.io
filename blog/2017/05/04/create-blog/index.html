hexo搭建博客

发表于 2017-05-04

前言

搭个博客，一来记录一些学习经验，二来锻炼下表达能力。

什么是hexo

  hexo 是一个快速、简洁且高效的博客框架。hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。

博客搭建

环境准备

  既然选择了要用hexo来搭建博客，那首先当然是看hexo的官方文档，了解hexo的操作使用流程。hexo官方中文文档。
  官方文档中有Node.js和git的安装步骤，由于本机（windows）已经安装过npm 以及 git 。所以就直接开始按照官方文档开始进行建站操作。
hexo官方提供了一个脚手架hexo-cli。通过npm全局安装脚手架。

hexo常用命令

新建文章

  新建一篇文章，如果没有设置 layout 的话，默认使用 _config.yml 中的 default_layout 参数（默认生成到source目录下的_post目录下)代替。如果标题包含空格的话，使用引号括起来。

清除缓存

  清除缓存文件(db.json)和已生成的静态文件(默认为public目录下的文件，可以更改到其他目录，后面会提到)。

生成静态文件

  hexo generate 的简写命令，用来生成静态文件。

启动静态服务

  hexo server的简写命令，用来开启一个服务器，对生成的静态文件进行热部署，并能监听文件变动(需要手动刷新浏览器)。默认端口4000，如果需要更改端口（比如4000端口被占用，改成5000)

部署

hexo deploy命令的缩写，可以将本地的文件推送到指定服务器上(比如github)。需要自己去配置_config.yml文件。

效果预览

先生成静态文件，再开启静态服务器

接着访问localhost:4000，即可看到效果。

更换主题

hexo 提供了多种主题样式供选择。我自己是选择了非常热门的next主题。可以到hexo主题列表里面去选择自己喜欢的主题。

找到主题对应的github仓库，然后把它clone下来到themes目录下。然后重新生成静态文件并开启服务器。

然后去hexo 的_config.yml文件中找到theme，并改成next。

另外next主题也提供了三种样式，默认主题样式是Muse，可以自行选择样式。

github page

github Pages可以被认为是用户编写的、托管在github上的静态网页。搭建的方式也很简单，只要创建一个名为(github用户名，你自己的github用户名).github.io的仓库就可以了。

部署到网站的子目录下

写完文章最后还是要部署到服务器上，我选择用github page来放置博客。不过问题是以前已经在该仓库下面放置了一些其他的小demo项目。如果就这么推上去的话，会和原有的项目冲突。
但是如果把整个public目录迁移到git page的仓库下，那每次写完文章又需要自己去copy到本地的git page的仓库下，就太麻烦了。想了想，觉得public目录肯定是可以配置的，找了找，果然在hexo根目录下的_config.yml中找到了。

设置后，博客在部署时就会放到git page的/blog/路径下了。但是默认的hexo生成的文件是以根目录进行引用的。当然这些都是可以在hexo根目录的_config.yml文件中进行配置的。
